1. Нужен текстовый редактор. [VS Code](https://code.visualstudio.com).

## Настройка сервера

1. Вам нужно зайти по директории

`\packages\ReweliseRP\system` и там будет файл **DB.js**. Открываем его с помощью заранее установленного текстового редактора.

Ищем там такие строчки как:

-   host
-   user
-   password
-   database

и изменяем данные в кавычках на свои. После чего сохраняем, и запсукаем **ragemp-server.exe** !

> После проделанных действий у Вас должен запуститься сервер и Вы сможете к нему подключиться!

-   armor - выдать себе бронежилет.
-   hp - вылечить себя

-   Включение полёта на кнопку F3.

## Клиентская часть (client_packages/)

### Основные компоненты:

1. **`admin/`**: Содержит скрипты для административных функций, таких как `esp.js`, `gm.js`, `inv.js`, `noclip.js`, `teleport.js`.
2. **`browser/`**: Ресурсы для CEF (Chromium Embedded Framework) браузера, используемые для отображения интерфейса.
    - **`components/`**: Внешние компоненты и библиотеки, такие как `vue-radial-progress` и `vue-simple-progress`.
    - **`systems/`**: Разделение по подсистемам, таким как `auth`, `autoschool`, `bank`, `houses`, `inventory`, `menu`, `jobs` и т.д.
    - **`index.html` и `index.js`**: Основные точки входа для CEF.
3. **`events/`**: Обработчики клиентских событий, такие как `events.js`, `hud.js`, `lsc.js`, `mission.js`, `rent.js`.
4. **`game_resources/`**: Ресурсы игры, такие как `dlcpacks/`.
5. **`utils/`**: Утилиты и вспомогательные функции, такие как `3dCamera.js`, `cef.js`, `death.js`, `methods.js`, `noclip.js`, `other.js`, `snake.js`, `zones.js`.
6. **`index.js`**: Основной файл инициализации клиента.

## Серверная часть (packages/ReweliseRP/)

### Основные компоненты:

1. **`commands/`**: Содержит серверные команды, такие как `admin.js`, `commands.js`, `esp.js`, `inv.js`, `teleport.js`.
2. **`events/`**: Серверные обработчики событий, такие как `events.js`, `mission.js`. Также есть подкаталог `basic/` с базовыми событиями, такими как `hud.js`.
3. **`json/`**: Конфигурационные файлы и данные в формате JSON, такие как `spawn_p.json`.
4. **`system/`**: Основные системные модули, такие как `DB.js` для взаимодействия с базой данных.
5. **`utilities/`**: Вспомогательные модули, такие как `death.js`, `items.js`, `time.js`, `weather.js`, `zones.js`.
6. **`index.js`**: Основной файл инициализации сервера.

## База данных (database/)

-   **`reweliserp.sql`**: SQL-скрипт для создания структуры базы данных.

## Рекомендации по добавлению новых файлов и папок

### Клиентская часть

#### Новые системы для CEF:

Если вы добавляете новую систему интерфейса (например, магазин), создайте новую папку в `browser/systems/` и разместите там необходимые HTML, CSS и JS файлы.

#### Новые утилиты:

Если потребуется новая функциональность, создайте соответствующий файл в `utils/`. Например, если нужно добавить обработку физики, можно создать `physics.js`.

#### Новые обработчики событий:

При добавлении нового игрового события создайте новый файл в `events/`, например, `newEvent.js`, и определите там соответствующую логику.

### Серверная часть

#### Новые команды:

Для новых административных или пользовательских команд добавьте соответствующий файл в `commands/`.

#### Новые серверные события:

Новые события следует добавлять в `events/`. Если они имеют специфический характер, можно создать подкаталог, как, например, `basic/`.

#### Новые системные модули:

Если нужна новая системная логика, добавьте соответствующий модуль в `system/`. Например, для управления экономикой игры можно создать `economy.js`.

#### Новые утилиты:

Утилиты, применяемые в серверной части, можно добавлять в `utilities/`.

### Общие рекомендации

-   **Именование файлов и папок**: Используйте осмысленные и описательные названия, чтобы сразу было понятно, за что отвечает данный модуль.
-   **Связь между клиентом и сервером**: Убедитесь, что все необходимые события и функции четко определены и правильно связаны через `mp.events.callRemote` для вызовов с клиента на сервер и `mp.events.call` для вызовов с сервера на клиент.
-   **Документирование**: Внутри файлов добавляйте комментарии и документацию для функций и модулей, чтобы облегчить поддержку и развитие проекта.
